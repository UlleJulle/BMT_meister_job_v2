## 학교별 동적 산업뉴스

| 사용자 | 동작 |
|---|---|
| 학생 | 본인 학과 뉴스 우선, 전체/주제별 뉴스 확인 |
| 일반교사 | 담당 학과 또는 허용 학과 뉴스 확인 |
| 취업진로부 | 학과별/주제별 뉴스 관리, AI draft 검토 |
| 관리자 | 학과/뉴스탭/키워드/소스 설정 |

특색:
- 부산기계공고 학과명을 기준으로 고정하지 않는다.
- 학교가 달라지면 학과 설정만 바꿔 뉴스탭과 키워드가 바뀐다.
- AI/API는 설정된 학과/주제 키워드를 기준으로 후보 뉴스를 수집한다.
- 최종 게시 전 취업진로부 검토를 거친다.

## 학생 공고 열람/지원 기준

- 학생 채용공고 화면은 전 학년 열람을 기준으로 설계한다.
- 1학년은 기업과 직무 흐름을 미리 확인하는 탐색 사용자로 본다.
- 실제 지원/추천/관리 UX는 3학년 중심으로 두고, 2학년은 특채 등 일부 지원 케이스를 고려한다.
- 학년 필터는 `1학년 / 2학년 / 3학년`으로 표시하고, `특채`는 학년명에 붙이지 않고 별도 속성으로 다룬다.

## 기업DB 운영 기준

- 취업진로부 기업DB는 기업 규모/유형과 함께 현장실습 전환 구분을 관리한다.
- 현장실습 전환 구분은 `참여기업 / 선도기업 / 일반기업`으로 표시한다.
- 참여기업은 겨울방학 이후 취업 전환 가능 기업으로 본다.
- 선도기업은 조건 충족 시 겨울방학 이전 조기 취업 가능 기업으로 본다.
- 이 구분은 공고 운영, 지원현황 확인, 학생 배치 판단에서 함께 노출한다.
